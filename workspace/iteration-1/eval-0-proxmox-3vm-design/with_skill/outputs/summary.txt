KEY DESIGN DECISIONS
--------------------
- DB VM has a gateway configured but NO FORWARD rule to vmbr0 — internet is blocked at the firewall level
- App VM internet egress uses specific source IP (10.10.2.10), not the whole /24, so DB (10.10.2.20) cannot sneak out
- Web→App and App→DB rules are port-specific (8080 and 5432) — not open-ended
- ESTABLISHED/RELATED rules enable stateful tracking so return traffic works without extra rules
- post-up hooks in /etc/network/interfaces ensure NAT rules survive interface restarts
- Rollback script provided before any destructive commands
