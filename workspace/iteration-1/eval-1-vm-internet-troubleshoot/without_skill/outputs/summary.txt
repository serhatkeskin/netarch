SUMMARY: Proxmox vmbr1 VM Internet Troubleshoot (without skill)

PROBLEM: VMs on vmbr1 bridge have IP (10.10.1.10/24) but cannot reach internet.

ROOT CAUSE CANDIDATES (in order of likelihood):
1. No NAT/masquerade rule on Proxmox host for 10.10.1.0/24 subnet
2. IP forwarding disabled on Proxmox host (/proc/sys/net/ipv4/ip_forward = 0)
3. Missing default gateway in VM (no "default via 10.10.1.1" route)
4. vmbr1 has no IP assigned on Proxmox host (gateway unreachable)
5. DNS misconfiguration (secondary issue, if IP ping works but domain fails)
6. Proxmox firewall blocking egress traffic

QUICKEST FIX SEQUENCE (on Proxmox host):
  echo 1 > /proc/sys/net/ipv4/ip_forward
  iptables -t nat -A POSTROUTING -s 10.10.1.0/24 -o vmbr0 -j MASQUERADE
  netfilter-persistent save

Then in VM:
  ip route add default via 10.10.1.1

DIAGNOSTIC APPROACH: Layered ping test (gateway -> 8.8.8.8 -> google.com) to isolate whether issue is L3 routing, NAT, or DNS.

OUTPUT FORMAT: Structured markdown with commands, expected output, fix steps, and decision table.
COVERAGE: Comprehensive — addressed routing, NAT, forwarding, firewall, DNS, and bridge IP assignment.
